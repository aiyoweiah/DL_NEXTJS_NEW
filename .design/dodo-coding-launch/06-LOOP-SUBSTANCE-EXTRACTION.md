# The Loop — substance extraction

**Date:** 2026-06-09
**Status:** Archival reference. Not a proposal.
**Purpose:** Capture what "The Loop" was doing for DODO Coding's brand and pedagogy, separately from its name. The user has retired The Loop from marketing surfaces (web, marketing guide, and the curriculum doc's "Loop emphasis" field) and will design a replacement later. This document is the source material for that future replacement — the brand-jobs and pedagogical structures it was carrying — so the substance is preserved even though the naming is gone.

> 📌 Read this **before** designing any DODO Coding methodology asset in the future. It catalogs what worked about The Loop, what was forced about it, and the brand-promise tensions it was sitting on top of. The replacement does not have to look like The Loop; it does have to do at least some of these jobs.

---

## 1. What The Loop was doing — the brand-jobs

Five distinct jobs. The replacement (whenever it lands) should be evaluated against this list — which of these does it carry, which does it drop on purpose, which does it carry differently?

### Job 1 — Naming the per-session rhythm

"Every session. Every week. In this order, without exception."

The brand wanted to claim that DODO Coding's sessions were *non-negotiably structured* — not improvised tutoring, not a curriculum that varies by Navigator mood, not a flexible "let's see what we get to today." The Loop gave the rhythm a name so the brand could point at it and say "this happens, every time."

**Why this job exists:** Parents of premium-paying audiences are buying *predictability of rigor.* A program that says "every session does X, Y, Z in order" outsells a program that says "every session is tailored." The fixed structure IS the product.

**Where it surfaces in DODO ELA's playbook:** §08 of the brand guide insists on confident, precise voice ("Lexile 650 to 820 in sixteen weeks"). The named-system rhythm is the structural sibling of those numbers — both deliver "we know exactly what will happen."

**Whether the replacement needs to do this job:** Yes. Without a named per-session rhythm, the brand reads as flexible, which reads as undisciplined, which reads as unworthy of the price. The new framing already preserves this through "every session" copy in `04-COPY-PASS.md` → `howItWorks.phases[1]` ("Three machine verbs. One human discipline. Every session."). The rhythm is there even though the name isn't.

### Job 2 — Encoding the machine / human split

Three machine verbs + one human verb, with the fourth visually distinguished. The brand promise rode on the closing fourth verb: *"…and to know when only a human should decide."*

**Why this job exists:** DODO Coding's reason for existing — its differentiation from "kids learn Python" / prompt engineering / AI camps — is the critical-thinking discipline that comes alongside the machine literacy. Without explicit "this part is the human's," DODO Coding becomes another coding school with a critical-thinking bullet point.

**What The Loop did well here:** It made the split *structural*, not advisory. The fourth verb wasn't a bonus feature; it was the apex of the four-part architecture.

**What was forced about it:** A four-verb sequence is the wrong shape if "critical thinking" is the discipline that runs *throughout* every session, not a phase that comes *after* the three machine phases. The Loop architecture put critical thinking at the end of a sequence; the curriculum (and the user's stated intent) puts it everywhere.

**Whether the replacement needs to do this job:** Yes, but probably as an *emphasis layered over the substance*, not as a phase at the end of a sequence. The current language-art + critical-thinking framing already does this — critical thinking is named as the discipline that anchors all three machine verbs, not as a fourth phase. The "machine / human" split is preserved without forcing it into a four-verb shape.

### Job 3 — Claiming brand-architecture parallel to DODO ELA

DODO ELA: Read → Think → Speak → Write.
DODO Coding: Read → Reason → Speak → Decide.

The parallel was the "one mind, two literacies" thesis made visual. The marketing guide called this the strongest brand-architecture move available — both products organized by the same four-phase shape, just applied to different language objects.

**Why this job mattered:** It implied that DODO Learning had a single coherent pedagogical theory that scaled across subject matter. Not just "we run two products"; rather, "we have a way of teaching, and it has the same shape in every product." That's a strong moat against competitors who only do one or the other.

**Whether the replacement needs to do this job:** **The user has explicitly retired this.** The "stop cross-crossing" decision (turn 12) and the language-art framing (turn 17) deliberately drop the parallel-asset claim. DODO Coding and DODO ELA are now positioned as *separate verticals* under one brand, not as parallel architectures. **The replacement should not try to re-establish this parallel.** If brand-architecture parallel is wanted later, it'll be wanted at a different layer (e.g., shared Navigator model, shared 16-week cycle, shared "we cite our sources" credibility — all of which already survive without the Loop).

### Job 4 — Functioning as the visual signature / OG image / iconic asset

The four-concentric-arc graphic was meant to be the brand's recognizable visual — the way the DODO ELA loop is. Marketing guide §3.2 called it "the single most marketable asset."

**Why this job exists:** Premium brands need a *visual hook* that travels across surfaces — homepage, social, OG cards, partner emails, conference slides. The Loop graphic was that asset.

**Whether the replacement needs to do this job:** Yes — DODO Coding still needs a visual signature, just not necessarily a four-phase one. Open question: what does the visual signature look like once the methodology is no longer a four-phase Loop? Candidates worth thinking about when the user revisits:
- The three-verb display itself, treated typographically as iconic (a la how some brands use a typeset motto as their primary visual identity)
- A pillar mark (one of the four open-frameworks logos elevated as the brand-association visual)
- A scholar-ink monogram or seal
- The `--ink` color treatment over a body-of-source text, as a visual register
- Something entirely new — the user's design call when revisiting

**What's in the staged docs in the meantime:** No visual signature. The `/coding` homepage uses typography and the four-pillar grid as its visual anchors, plus the `--ink` color treatment on the critical-thinking heading. This works as a launch placeholder but is not a brand asset of the kind the Loop graphic was supposed to be.

### Job 5 — Curriculum-design vocabulary

Each cycle in `program-streams-v1.md` had a "Loop emphasis" field — *"Read + Reason (introduce all four phases, deepest in Read+Reason)."* This let the curriculum designer signal which phases a cycle leaned into.

**Why this job existed:** Useful internal taxonomy for sequencing — week 12 of Intermediate I3 emphasized the Reason phase deepest; week 14 of Advanced A3 emphasized the Decide phase. The field gave the curriculum doc a way to track session-level pedagogy across the multi-year arc.

**User's clarification (turn 25):** The Loop is a marketing term, *not* a Navigator-essential one. So this field was a marketing artifact in the curriculum doc, not a real Navigator-side need.

**Whether the replacement needs to do this job:** No. The "Loop emphasis" field can be removed from the curriculum doc. If a Navigator wants to know "what does this cycle emphasize?" the cycle's existing fields (Theme, Topics, AI literacy thread, Primary pillar, Capstone) already carry that information without needing a phase-tag.

## 2. Pedagogical substance that survives outside the Loop architecture

The Loop named these things and bundled them into a four-phase sequence. The naming is retired; the substance survives, just unbundled:

| Substance element | Preserved how, in the current direction |
|---|---|
| AI4K12 Big Idea 1 (Perception) | Surfaces as the **Read** machine verb in `whatWeTeach`. |
| AI4K12 Big Ideas 2 + 3 (Reasoning + Learning) | Surface as the **Think** machine verb. |
| AI4K12 Big Idea 4 (Natural Interaction) | Surfaces as the **Write** machine verb. |
| AI4K12 Big Idea 5 (Societal Impact) | Surfaces as **critical thinking**, the human discipline layered throughout. |
| The four open-source pillars (AI4K12, CMU CS Academy, Huyen AI Engineering, Claude/Gemini ambient lab) | Survive as the `pillars` section on `/coding/home` and `/coding/methodology`. |
| The per-session in-this-order rhythm | Survives as copy ("every session" / "throughout every session") without being a named methodology. |
| The 16-week cycle structure | Unchanged — `/coding/program` carries this. |
| The Navigator model + Socratic dialogue + 1-on-1 | Unchanged — shared with ELA. |

| Glass-box AI literacy | Surfaces implicitly in the machine-verb descriptions ("how AI perceives," "how AI processes," "how AI generates" — not "how to use AI"). |

Everything the Loop was carrying pedagogically is alive in the staged copy. The branding around it is what's been retired.

## 3. Brand-promise structures the Loop was sitting on

The marketing guide v1.0 brand promise reads:

> *DODO Learning ELA teaches a child to think precisely in human language.*
> *DODO Coding teaches a child to think precisely in machine language — and to know when only a human should decide.*

Structurally this is:
- **Premise** (ELA's value)
- **Parallel premise** (Coding's value — leans on the parallel claim, Job 3 above)
- **Differentiation hook** (the fourth-verb closing, Job 2 above)

With the parallel claim retired (Job 3) and the fourth-verb structure retired (Job 2), the brand promise needs a new shape. The current staged direction reframes it as:

> *DODO Coding is a language art for the AI age. We teach how AI reads, thinks, and writes — and how to think critically about it.*

Structurally this is:
- **Category claim** ("a language art")
- **Substance** ("how AI reads, thinks, and writes")
- **Human discipline** ("how to think critically about it")

Three moves instead of three different three moves. Lower brand-architecture ambition but higher honesty. The replacement, when it comes, should know what brand-promise structure it's serving — and whether it wants to restore the parallel-with-ELA move (which the user has currently chosen to drop).

## 4. What this leaves on the table for the future replacement

When the user designs the replacement (per their note "we will come up with something different down the road"), the open question set is:

1. **Does it have a name?** The current direction has no named methodology. A future replacement could reintroduce one — but it should be earned by carrying brand-jobs the unnamed version can't, not added because "DODO ELA has LCS, so DODO Coding should have one too."

2. **Does it have a visual signature?** (Job 4.) If yes, what shape — typographic, iconic, pillar-anchored, color-anchored, monogram, or something else?

3. **Does it restore the parallel-with-ELA claim?** (Job 3.) The user has dropped this. A future replacement could restore it at a different layer (not Loop-vs-Loop; could be Navigator-model-vs-Navigator-model, or 16-week-vs-16-week, or measurement-cadence-vs-measurement-cadence). But it doesn't have to.

4. **Does it stay critical-thinking-as-emphasis, or move critical-thinking back into a phase?** Current direction: emphasis. The future replacement could move it back to a phase if a four-verb structure earns its weight again, but the user's stated intent is that critical thinking is *throughout*, not after.

5. **Does it name the machine / human split visibly?** Currently the split is in the copy ("how AI reads, thinks, and writes" vs "how your child thinks critically about it"), not in a graphic. A future replacement might surface the split graphically — but that opens up the "fourth phase" temptation again, so design carefully.

6. **Does it survive into the curriculum doc?** The Loop's curriculum role (Job 5) was a marketing import, not a Navigator need. A future named methodology might or might not need a corresponding curriculum-side concept. Default: assume not, unless Navigators ask for it.
